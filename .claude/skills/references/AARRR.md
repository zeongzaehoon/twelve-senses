# AARRR (Pirate Metrics) 방법론

## 개요
Dave McClure(500 Startups)가 제안한 스타트업 성장 지표 프레임워크.
사용자 여정의 5단계를 측정해 어디서 누수가 일어나는지 파악.

## 5단계 퍼널

```
Acquisition → Activation → Retention → Referral → Revenue
  획득           활성화        유지          추천        수익
```

### 각 단계 정의 및 Twelve Senses 적용

| 단계 | 정의 | 핵심 지표 | Twelve Senses 적용 |
|------|------|----------|-------------------|
| **Acquisition** | 어떻게 유입되나? | CAC, 채널별 유입 수, 전환율 | 크펀 유입, SNS 유입, 앱스토어 검색 |
| **Activation** | 첫 경험이 만족스러웠나? | 첫 핵심 행동 완료율 | 첫 AI 레시피 생성 완료 |
| **Retention** | 계속 돌아오나? | D7/D30 재방문율, Churn Rate | 주간 능동 사용률, 구독 갱신율 |
| **Referral** | 자발적으로 추천하나? | NPS, 바이럴 계수(K-factor) | 레시피 공유 수, 초대 전환율 |
| **Revenue** | 수익이 발생하나? | ARPU, LTV, MRR | 하드웨어 판매 + 구독 + B2B |

## 병목 구간 진단법

```
각 단계 전환율 계산:
Activation Rate = 활성화 완료 / 신규 가입
Retention Rate = D30 복귀 / 첫 활성화
Referral Rate = 추천 발생 / 활성 사용자

가장 낮은 전환율 = 현재 병목 → 이 단계 집중 개선
```

## Twelve Senses AARRR 목표 (초기)

| 단계 | 현재 | 목표 (크펀 후 3개월) | 개선 방법 |
|------|------|---------------------|----------|
| Acquisition | 0 | 유입 1,000명/월 | SNS 콘텐츠, 크펀, 바이럴 |
| Activation | - | 첫 레시피 생성율 70% | 온보딩 플로우 최적화 |
| Retention | - | D30 재방문율 50% | 맞춤 알림, 새 레시피 추천 |
| Referral | - | K-factor 0.3 이상 | 레시피 공유 기능, 친구 초대 인센티브 |
| Revenue | 0 | MRR 500만원 | 구독 전환 캠페인 |

## AARRR vs Growth Loops
AARRR = 선형 퍼널 (한 방향 흐름)
Growth Loops = 순환 시스템 (출력이 다시 입력으로)
→ 두 개를 병행해 단기 퍼널 최적화 + 장기 복리 성장 설계
