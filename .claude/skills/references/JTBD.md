# Jobs-to-be-Done (JTBD) 방법론

## 개요
고객은 제품을 사는 게 아니라 "일(Job)"을 해결하기 위해 제품을 "고용"한다는 이론.
Clayton Christensen이 발전시키고 Tony Ulwick의 ODI(Outcome-Driven Innovation)로 실용화됨.

## 핵심 원칙
- **Jobs는 안정적이다**: 기술·제품은 변해도 고객이 해결하려는 근본 목적은 불변
- **제품이 아닌 Job에 집중**: "더 좋은 드릴"이 아니라 "벽에 구멍을 뚫는 일"
- **경쟁 대안은 넓다**: 직접 경쟁사뿐 아니라 고객이 현재 쓰는 모든 대안 포함

## Job 유형 3가지

| 유형 | 설명 | 예시 (향기 제품) |
|------|------|----------------|
| **기능적 Job** | 실용적 목적 달성 | "공간에 향기를 채운다" |
| **감성적 Job** | 개인의 감정 충족 | "집에 돌아왔을 때 심리적 안정감" |
| **사회적 Job** | 타인에게 보여지는 자아 | "취향 있는 사람으로 인식되고 싶다" |

## 4 Forces 프레임워크 (전환 분석)

```
         [Push]                    [Pull]
현재 상황의 불만 ──────→ 새 제품으로 전환 ←────── 새 솔루션의 매력
         
         [Anxiety]               [Inertia]
    새 제품에 대한 불안 ←────── 현재 습관·관성
```

- **Push**: 현재 대안(캔들, 기존 디퓨저)의 불만족 포인트
- **Pull**: Twelve Senses가 제공하는 새로운 가치
- **Anxiety**: 신제품 도입 우려 (가격, 복잡성, 신뢰성)
- **Inertia**: 기존 습관 유지 동기

## JTBD 인터뷰 5가지 핵심 질문

1. **언제** 이 제품을 "고용"하는가? (상황·맥락·트리거)
2. **진짜 목적**은? (기능적 / 감성적 / 사회적 Job)
3. **현재 대안**은? (무엇으로 해결하고 있나)
4. **전환 동기**는? (기존 대안의 어떤 점이 불만족인가)
5. **성공 정의**는? ("완벽히 해결됐다"는 언제인가)

## Twelve Senses JTBD 적용

**핵심 Job**: "내 공간을 나만의 감각 세계로 만들고 싶다"
- 기능적: 원하는 향기 조합을 정밀하게 제어
- 감성적: 귀가했을 때 즉각적인 심리적 전환
- 사회적: 향기 취향이 있는 사람으로 자기 표현

**주요 Unmet Needs**:
- 기존 디퓨저: 조향 불가, 향 강도 조절 어려움
- 캔들: 방치 불가, 화재 위험, 지속성 부족
- 아로마 에센셜: 직접 혼합 번거로움, 일관성 없음

## 출력 형식
```
### JTBD 분석
**핵심 Job**: [한 문장]
**Job 유형**: [기능적/감성적/사회적]
**현재 대안 & 불만**: [리스트]
**전환 동기**: [Push/Pull 각 2~3개]
**Success Metrics (고객 관점)**: [측정 가능한 결과 3개]
```
