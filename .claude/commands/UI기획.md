---

## 방법론 1: Design Thinking — 5단계 프로세스
| 단계 | 핵심 질문 |
|------|----------|
| **공감(Empathize)** | 실제 사용자는 어떤 상황에서 앱을 여는가? 무엇이 불편한가? |
| **문제 정의(Define)** | "우리는 [누구]가 [무엇]을 할 수 있도록 [어떻게] 해야 한다" |
| **아이디어 발산(Ideate)** | HMW(How Might We) 질문 3개 도출 → 각각 아이디어 3개 |
| **프로토타입(Prototype)** | 이번 주 만들어볼 수 있는 가장 작은 UI 실험 |
| **테스트(Test)** | 무엇을 검증할 것인가? 성공 기준? |

## 방법론 2: User Story Mapping — UX 플로우
Twelve Senses 앱의 핵심 사용자 여정을 매핑:

```
[사용자 활동] 온보딩 → 향기 설정 → 레시피 선택 → 디퓨저 제어 → 히스토리 확인
      ↓
[필수 스텝] 각 활동별 핵심 단계
      ↓
[MVP 라인] — — — — — — — — — — — — — — — — — — —
      ↓
[다음 릴리즈] 추가 기능
```

각 스텝에서: 무엇이 불명확한가? 어디서 이탈할 것 같은가?

## 방법론 3: Lean UX — 가정-실험-학습
| 가정(Assumption) | 가장 작은 실험 | 성공 지표 |
|-----------------|--------------|----------|
| 사용자가 레시피를 저장한다 | 저장 버튼 A/B 테스트 | 저장율 > X% |
