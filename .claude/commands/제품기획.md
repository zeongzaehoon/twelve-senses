**시작 전**: Bash로 `date +%Y-W%V`를 실행해 현재 주차를 확인해. 이후 모든 경로의 `$(date +%Y-W%V)` 자리에 실제 값을 사용해.

CLAUDE.md의 Twelve Senses 프로젝트 컨텍스트를 읽고, `reports/$(date +%Y-W%V)/SWOT분석.md`가 있으면 읽어 강점·약점 기반으로 기능 우선순위를 보정해. WebSearch로 경쟁 스마트 디퓨저·AI IoT 제품 동향을 조사한 뒤, 아래 3가지 방법론으로 **제품기획 보고서**를 작성해줘.

---

## 방법론 1: Jobs-to-be-Done (JTBD)
고객이 제품을 "고용"하는 진짜 이유를 파악해:
- **Functional Job**: 고객이 기능적으로 달성하려는 것 (예: 공간에 향기 채우기)
- **Emotional Job**: 고객이 느끼고 싶은 감정 (예: 나만의 감성 공간 소유)
- **Social Job**: 타인에게 어떻게 보이고 싶은지 (예: 취향 있는 사람)
- 현재 고객이 쓰는 대안 솔루션과 전환 비용
- 성공/실패의 정의

## 방법론 2: RICE Scoring — 기능 우선순위
후보 기능/이슈를 아래 공식으로 점수화:

**RICE Score = (Reach × Impact × Confidence) / Effort**

| 기능 | Reach(주간 영향 유저) | Impact(1~5) | Confidence(%) | Effort(인주) | RICE Score |
|------|-----------------------|-------------|---------------|-------------|------------|

상위 5개 선정 후 우선 진행 근거 작성.

## 방법론 3: MoSCoW — MVP 경계 정의
| 분류 | 기준 |
|------|------|
| **Must-have** | 없으면 제품 불가 |
| **Should-have** | 중요하지만 초기 출시 후 가능 |
| **Could-have** | 있으면 좋은 것 |
| **Won't-have** | 이번 범위에서 제외 |

---

## 출력 형식
결과를 `reports/$(date +%Y-W%V)/제품기획.md`에 저장. 구성:
1. JTBD 분석 (기능적·감성적·사회적 Job)
2. RICE 우선순위 테이블 (Top 5)
3. MoSCoW 분류표
4. 이번 스프린트 백로그 제안
5. 경쟁 제품 기능 벤치마킹 (WebSearch 기반)
6. 제품 KPI 목표
