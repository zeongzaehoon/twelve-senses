**시작 전**: Bash로 `date +%Y-W%V`를 실행해 현재 주차를 확인해.

CLAUDE.md의 Twelve Senses 프로젝트 컨텍스트를 읽고, `reports/$(date +%Y-W%V)/브랜딩전략.md`가 있으면 읽어 브랜드 아이덴티티 방향을 반영해. WebSearch로 하이엔드 테크·라이프스타일 디자인 트렌드를 조사한 뒤, 아래 3가지 방법론으로 **디자인 방향 보고서**를 작성해줘.

---

## 방법론 1: Design Sprint 핵심 질문 (압축 버전)
5일 스프린트를 주간 단위로 압축해 진행할 핵심 의사결정:
- **Map**: 이번 주 해결할 가장 중요한 디자인 문제는?
- **Sketch**: 가능한 솔루션 아이디어 3가지
- **Decide**: 1개 선택 + 선택 근거
- **Prototype**: 이번 주 만들 수 있는 최소 산출물
- **Test**: 누구에게, 무엇을, 어떻게 검증?

## 방법론 2: Atomic Design — 디자인 시스템 점검
| 레벨 | 내용 | 현황 점검 |
|------|------|----------|
| **Atoms** | 색상, 타이포그래피, 아이콘, 간격 | 정의됨 / 미정의 / 개선 필요 |
| **Molecules** | 버튼, 인풋, 카드 등 기본 컴포넌트 | |
| **Organisms** | 향기 컨트롤러, 레시피 카드, 오일 잔량 UI | |
| **Templates** | 주요 화면 레이아웃 | |
| **Pages** | 실제 콘텐츠가 채워진 최종 화면 | |

## 방법론 3: Brand Identity — 시각 언어 정의
하이엔드 Tech Object 포지셔닝 기준으로:
- **색상 팔레트**: 주색·보조색·강조색, 감성 연상 어휘
- **타이포그래피**: 한글/영문 서체, 위계 규칙
- **모션·인터랙션**: 향기 확산을 표현할 애니메이션 방향
- **하드웨어 디자인 언어**: 15×15×15cm 폼팩터 스타일 방향 (29cm·EQL 입점 기준)
- **일관성 점검**: 앱 ↔ 하드웨어 ↔ 패키징 ↔ 마케팅 소재

## 출력 형식
결과를 `reports/$(date +%Y-W%V)/디자인.md`에 저장. 구성:
1. Design Sprint 이번 주 의사결정 요약
2. Atomic Design 시스템 현황표 + 개선 필요 항목
3. Brand Identity 가이드라인 점검
4. 경쟁 제품 디자인 벤치마킹 (WebSearch: Dyson, 샤오미 등)
5. 이번 주 디자인 작업 우선순위 Top 5

## 마지막 단계: Notion 저장
로컬 저장 완료 후 Read 도구로 `.claude/skills/Notion저장.md`를 읽고, 해당 보고서 유형 리치 포맷 템플릿을 적용해 Notion에 저장:
- Executive Summary (3줄)
- 핵심 인사이트 Top 5
- 이번 주 실행 과제 Top 3
