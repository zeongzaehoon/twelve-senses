# 공통 분석 규칙

모든 주간 분석 커맨드(`/주간분석`, `/시장조사`, `/PEST분석` 등)에 적용되는 기본 규칙.

## 시작 전 필수 확인

1. **주차 확인**: 반드시 `date +%Y-W%V` 실행해 실제 주차 확인
2. **프로젝트 컨텍스트**: CLAUDE.md 읽어 최신 프로젝트 상태 파악
3. **기존 리서치 활용**: `reports/$(date +%Y-W%V)/_리서치.md` 있으면 먼저 읽기 → 중복 WebSearch 금지

## 데이터 수집 규칙

- **WebSearch 재사용**: 동일 주차 내 이미 수집된 데이터는 `_리서치.md`에서 참조
- **출처 필수**: 모든 수치·데이터에 URL 출처 포함
- **추정값 표시**: 실제 데이터가 없는 경우 `[추정]` 명시
- **최신 데이터 우선**: 시장 수치는 발행 연도 반드시 표기

## 보고서 저장 규칙

### 로컬 저장 (필수)
```
reports/$(date +%Y-W%V)/[보고서명].md
```

### Notion 저장 (필수)

모든 분석 보고서는 로컬 저장 후 Notion에도 저장한다.

> ⚠️ **Notion 저장 전 필수 선행 작업**:
> Notion 저장 단계에 진입하기 전에 반드시 아래 두 파일을 Read 도구로 읽어야 한다.
> 읽지 않고 저장하면 Notion 페이지가 깨진 포맷으로 생성되므로 규칙 위반이다.
>
> 1. **Read** `.claude/skills/Notion저장.md` — 저장 절차 + 보고서 유형별 리치 포맷 템플릿
> 2. **Read** `.claude/skills/references/Notion-블록레퍼런스.md` — 블록 타입 JSON 예시

**Notion 저장 내용**:
- DB 속성: Executive Summary(3줄), 핵심 인사이트 Top 5, 실행 과제 Top 3
- 페이지 본문: `Notion저장.md`의 보고서 유형 템플릿 그대로 적용
  (callout, table, column_list, image 차트, toggle 등 리치 포맷 사용 필수)

## 품질 기준

- 분석표 빈칸 없이 채우기 (데이터 없으면 `[추정]` + 근거)
- 지난 주 보고서 대비 변화 언급
- 실행 가능한 액션 아이템 포함 (담당·기한 명시)
- 각 분석에서 SWOT의 S/W/O/T 중 해당 항목 추출해 명시

## 보고서 연계 규칙

```
_리서치 → 시장조사·PEST·고객경쟁사 → SWOT → 전략기획·제품기획 → UI기획·디자인·브랜딩 → 마케팅
```

- 상위 단계 보고서가 있으면 반드시 읽고 반영
- 하위 단계가 상위 단계에 모순되면 반드시 언급

## 에이전트 활용 규칙

복잡한 분석은 전문 에이전트에게 위임:
- 시장·경쟁 분석 → `시장조사분석가`, `경영전략분석가`
- 마케팅·브랜딩 분석 → `마케팅분석가`, `브랜딩분석가`
- 영업·B2B 전략 → `영업전략분석가`

에이전트 위임 시 충분한 컨텍스트 (프로젝트 배경 + 이번 주 데이터) 함께 전달.
