## 1. Design Thinking 5단계

### 1-1. Empathize — 사용자 핵심 페인포인트

> 고객경쟁사분석.md Empathy Map 기반. 타겟: MZ세대 B2C 핵심 고객 (25~35세, 취향 소비자)

**페인포인트 #1 — "레이어링 시행착오가 무섭다"**

Empathy Map의 Thinks 항목에서 확인된 핵심 심리: *'근데 어떻게 섞어야 좋은지 모르겠고 시행착오가 무서워.'* 향 조합 실패에 따른 비용·낭비 경험이 레이어링 시도 자체를 억제한다. 현재 대안(여러 브랜드 디퓨저 교체, 향수 레이어링)은 결과 예측이 불가능하고 실패 비용이 높다. JTBD #2의 전환 동기 "레이어링 결과 예측 불가"와 직결된다.

**페인포인트 #2 — "내 취향을 정확히 표현할 향을 아직 못 찾았다"**

Empathy Map의 Thinks 항목: *'내 취향을 정확히 표현할 향을 아직 못 찾았어.'* 향기 제품 구매 전 성분·리뷰를 꼼꼼히 확인하는 Does 행동이 이 불안을 방증한다. 향 선택 실패 시 낭비감·아쉬움(Feels)이 반복되고, 셀프 선물 소비에도 불구하고 만족도는 낮다. AI 콜드 스타트 문제(제품기획.md SWOT W5)가 UI 설계에서 온보딩으로 반드시 해소되어야 한다.

**페인포인트 #3 — "자동화에 기대가 있지만 작동 원리를 모르면 불안하다"**

Empathy Map Thinks: *'AI가 내 취향을 학습해준다면 진지 편할 것 같은데 그게 가능해?'* Feels: *AI·자동화에 대한 기대와 미지의 불안 공존.* 스마트홈 연동 니즈가 있으나(Says), AI가 왜 이 향을 추천했는지 설명이 없으면 신뢰도가 낮다. JTBD #5 "성분 투명성"에 대한 니즈와 연결되며, 추천 근거를 UI 위에서 가시화하는 것이 중요하다.

### 1-2. Define — HMW (How Might We) 질문

**HMW #1 — "어떻게 하면 레이어링 초보도 자신 있게 나만의 향을 만들 수 있을까?"**

페인포인트 #1에서 도출. 향 조합 실패 공포를 제거하는 것이 핵심이다. AI 가이드가 시행착오를 대신하고, 결과를 미리 언어로 묘사해 줌으로써 확신을 부여해야 한다.

**HMW #2 — "어떻게 하면 AI가 나를 알아간다는 느낌을 즉시, 그리고 지속적으로 줄 수 있을까?"**

페인포인트 #2에서 도출. 첫 온보딩부터 "내 취향이 반영됐다"는 즉각적 피드백이 필요하다. 사용이 누적될수록 레시피 품질이 올라간다는 것을 사용자가 체감하게 해야 한다.

**HMW #3 — "어떻게 하면 AI 추천의 이유를 투명하게 보여줘 신뢰를 쌓을 수 있을까?"**

페인포인트 #3에서 도출. "이 향을 추천한 이유"를 한 문장으로 카드에 표시하고, RFID 오일 성분 정보를 추천 맥락 안에 배치하면 불안이 줄어든다.

### 1-3. Ideate — HMW별 솔루션 아이디어

**HMW #1 솔루션 아이디어**

| # | 아이디어 | 개요 |
|---|---------|------|
| I-1 | 향 취향 인터뷰 온보딩 | 첫 실행 시 5~7개 질문(선호 분위기, 계절, 감정 키워드)으로 AI 레시피 3종 즉시 생성 |
| I-2 | "이 향은 ~같은 느낌이에요" 묘사 카드 | 레시피마다 후각이 아닌 언어(감각, 분위기)로 결과 설명 — 선택 자신감 부여 |
| I-3 | 슬라이더 믹싱 UI | 4채널 오일 비율을 슬라이더로 직접 조절, 변경할 때마다 AI가 향 묘사 실시간 업데이트 |
| I-4 | 비슷한 향 추천 ("이것도 좋아할 것 같아요") | 현재 레시피에서 한 채널만 바꾸는 "변주 추천" — 실패 부담 없이 탐색 유도 |
| I-5 | 레시피 미리 이름 붙이기 | 저장 전 "이 향의 이름을 지어주세요" 입력창 → 소유감·애착 형성 |

**HMW #2 솔루션 아이디어**

| # | 아이디어 | 개요 |
|---|---------|------|
| I-6 | "나의 향 DNA" 레이더 차트 | 플로럴·우디·시트러스·머스크·스파이시 5축 레이더 차트로 취향 프로파일 시각화 |
| I-7 | 레시피 아카이브 + 누적 카운터 | "내가 만든 레시피 N개" 메인 화면 표시 — 사용 기간이 쌓이는 느낌 |
| I-8 | 계절별 개인화 알림 | "지난 봄에 가장 좋아했던 향, 올 봄에도 준비할까요?" 타이밍 푸시 |
| I-9 | 사용 히스토리 타임라인 | 날짜·날씨·향·시간이 카드 형태로 쌓이는 나의 향기 기록 뷰 |

**HMW #3 솔루션 아이디어**

| # | 아이디어 | 개요 |
|---|---------|------|
| I-10 | "오늘 추천 이유" 한 줄 배지 | 날씨 API 기반 추천 시 "오늘 흐리고 습해서 우디 계열을 추천했어요" 카드 상단 표시 |
| I-11 | RFID 성분 인포그래픽 | 카트리지 장착 시 팝업으로 원산지·주요 성분·인증 마크 시각화 |
| I-12 | 추천 점수 투명 표시 | "취향 일치도 87%" 숫자와 함께 일치 근거 키워드 3개 노출 |
| I-13 | 안심 모드 표시 레이어 | 자녀·반려동물 모드 활성화 시 농도 제한 범위를 화면 상단 배너로 항상 표시 |

### 1-4. Prototype — 핵심 화면 3개 와이어프레임 설명

> 텍스트 와이어프레임. 화면당 레이아웃 구조, 핵심 컴포넌트, 인터랙션 흐름을 기술.

**[화면 1] 홈 화면**

```
┌─────────────────────────────┐
│  [상태바]                    │
│  ────────────────────────── │
│  좌상단: 디바이스 연결 상태 아이콘 + 기기명
│  우상단: 프로필 아이콘 (마이페이지)
│                              │
│  ──[ 오늘의 날씨 추천 카드 ]──  │
│  📍 서울 / 흐림 13°C           │
│  "습한 날엔 우디 향이 잘 어울려요" │
│  [레시피 이름] ▶ 지금 발향하기   │
│  ──────────────────────────  │
│                              │
│  ──[ 발향 제어 패널 ]──        │
│  ●●●●○ 강도 4단계              │
│  [ON / OFF 토글 — 대형 버튼]   │
│  발향 중 · 오일 잔량 표시 바     │
│  ──────────────────────────  │
│                              │
│  ──[ 내 레시피 빠른 실행 ]──    │
│  [카드 1: 월요일 아침]          │
│  [카드 2: 퇴근 후 릴렉스]       │
│  [카드 3: + 새 레시피]          │

│                              │
│  하단 탭바: 홈 / 레시피 / 오일 / MY │
└─────────────────────────────┘
```

**핵심 인터랙션**:
- 오늘의 날씨 추천 카드의 "지금 발향하기" 탭 → 1초 이내 디바이스 발향 실행 (오버슈팅 알고리즘 연동)
- ON/OFF 토글: 발향 중일 때 펄스 애니메이션으로 시각 피드백
- 빠른 실행 카드: 좌우 스크롤, 탭 즉시 실행 (확인 팝업 없음 — 마찰 최소화)
- 오일 잔량 바: 20% 이하 시 주황색 + 탭하면 오일 관리 화면 이동

**설계 근거**: JTBD #1 "귀가 2분 내 원하는 향이 퍼지는 경험" + RICE Score 최상위 기능인 원터치 발향(Score 285.0)을 홈의 최상위 레이어에 배치. 날씨 추천 카드는 HMW #3 솔루션 I-10 적용.

### 1-5. Test — 검증 질문 및 성공 기준

**검증 질문 1 — 온보딩 레시피 생성이 AI에 대한 신뢰를 형성하는가?**

- 방법: 첫 사용 테스트 (n=10, 25~35세 MZ세대) — 온보딩 완료 후 "이 레시피가 내 취향에 맞는다고 느끼나요?" 5점 척도
- 성공 기준: 평균 4.0점 이상, "레시피 3종 중 하나 이상 즉시 발향" 선택률 70% 이상

**검증 질문 2 — 직접 만들기 슬라이더 UI에서 레이어링 초보가 완주하는가?**

- 방법: 비관찰 사용성 테스트 (n=8) — 슬라이더 조작부터 레시피 저장까지 완주율 측정 + Think Aloud
- 성공 기준: 완주율 75% 이상, 중도 포기 이탈 지점 미발생

**검증 질문 3 — 오일 잔량 알림이 재구매 전환으로 이어지는가?**

- 방법: Q3 크라우드펀딩 후원자 대상 A/B 테스트 — 알림 없는 그룹 vs. 알림 있는 그룹
- 성공 기준: 알림 수신 후 48시간 내 재주문 전환율 30% 이상

## 2. 핵심 화면 흐름 — 주요 3개 화면 와이어프레임 설명

> 위 1-4 섹션에서 화면 3개 와이어프레임을 상세 기술함. 본 섹션은 화면 간 내비게이션 흐름을 정리.

### 2-1. 주요 화면 내비게이션 맵

```
[스플래시 / 온보딩]
        │
        ├─ 최초 실행: 향 취향 인터뷰 (5~7 질문) → AI 레시피 3종 생성 → 홈
        └─ 재실행: 홈 화면
                │
       ┌────────┼────────┬──────────┐
       ▼        ▼        ▼          ▼
    [홈]    [레시피]    [오일]      [MY]
       │        │        │          │
  발향 제어  레시피 생성  카트리지 상태  프로필/향 DNA
  날씨 추천  AI 추천탭   잔량 표시    레시피 아카이브
  빠른 실행  직접만들기   RFID 성분   히스토리 로그
             저장/편집   구독 관리   설정/안심 모드
```

### 2-2. 핵심 사용자 플로우 — 퇴근 후 발향 시나리오 (JTBD #1)

```
귀가 → 앱 열기 → 홈 화면
  → 오늘의 날씨 추천 카드 확인 ("흐린 날, 우디 향 추천")
  → "지금 발향하기" 탭 (원터치)
  → 디바이스 발향 시작 (≤500ms 반응, 오버슈팅 알고리즘)
  → 발향 중 상태 피드백 (홈 화면 펄스 애니메이션)
```

### 2-3. 핵심 사용자 플로우 — AI 레시피 생성 시나리오 (JTBD #2)

```
레시피 탭 → AI 추천 탭 선택
  → 감정 칩 선택 ("편안한")
  → 레시피 3종 카드 로딩 (≤10초)
  → 카드 탭: 취향 일치도 + 추천 이유 확인
  → "지금 발향하기" 또는 "저장하기" 선택
  → 이름 입력 ("퇴근 후 릴렉스") → 저장
  → 홈 빠른 실행 카드에 자동 추가
```

### 2-4. 핵심 사용자 플로우 — 오일 잔량 관리 시나리오 (JTBD #5)

```
푸시 알림 수신 ("베르가못 오일 22% 남았어요")
  → 알림 탭 → 오일 관리 화면 이동 (딥링크)
  → 슬롯 2 카드 확인: 잔량 바 + 경고
  → "바로 구독 주문하기" 탭
  → 인앱 커머스 페이지 (Should Have, Q3 추가)
  또는 RFID 성분 인증 배지 탭 → 성분 PDF 확인 → 안심
```

## 4. Lean UX 가정-실험 테이블

> 가장 리스크가 높은 가정 3개를 선정. 리스크 수준 기준: **확신도 낮음 + 실패 시 사업 임팩트 높음**.

| 가정 | 리스크 수준 | 검증 방법 | 성공 기준 | 학습 액션 |
|------|-----------|---------|---------|---------|
| **[가정 1] "AI 레시피 추천을 받으면 사용자는 직접 만들기보다 AI 추천을 더 신뢰하고 선택할 것이다"** — AI 추천 기능이 핵심 차별화이자 사용 패턴의 전제이지만, 실제 사용자가 AI 결과를 신뢰할지 검증되지 않음. 신뢰하지 않으면 AI 엔진 개발 투자 전체가 흔들림. | **높음** (확신도 낮음, 실패 시 핵심 UVP 붕괴) | **Fake Door 테스트**: 프로토타입 앱에서 AI 추천 탭과 직접 만들기 탭을 모두 제공. 사용자 10명 대상 첫 레시피 생성 세션 관찰 — 어느 탭을 먼저 클릭하고, AI 추천 결과를 수용하는지 비율 측정. 추가로 추천 결과 만족도 5점 척도 설문. | AI 추천 탭 선택률 60% 이상 / AI 추천 레시피 수용(발향 또는 저장) 비율 50% 이상 / 추천 만족도 평균 3.8점 이상 | 성공 → AI 추천 UI를 홈 화면 최상위에 유지 / 실패 → 직접 만들기를 기본 뷰로 전환하고 AI 추천은 "제안" 방식으로 서브 포지셔닝. 추천 이유 텍스트(HMW #3 I-10) 강화 재설계. |
| **[가정 2] "잔량 알림을 받은 사용자는 48시간 내 구독 재주문으로 전환할 것이다"** — RFID 기반 잔량 알림이 구독 Lock-in의 핵심 전환 트리거로 설계되어 있으나, 알림→구매 전환율이 낮으면 RFID 인프라 투자 대비 수익 모델이 작동하지 않음. | **높음** (수익 모델 직결, 확신도 중간) | **A/B 테스트**: Q3 크라우드펀딩 후원자 50명 대상. 그룹 A: 잔량 20% 알림만 발송. 그룹 B: 잔량 20% 알림 + "지금 주문하면 다음 발송까지 딱 맞아요" 문구 + 원클릭 주문 딥링크. 각 그룹의 48시간 내 주문 전환율 및 평균 주문 금액 비교. | 그룹 B 전환율 30% 이상 / 그룹 A 대비 전환율 1.5배 이상 / 주문 완료까지 화면 이탈 없이 2탭 이내 완료 비율 70% 이상 | 성공 → 알림 문구 + 딥링크 패턴을 표준화하고 계절별 알림에도 동일 패턴 적용 / 실패 → 알림 타이밍(20% vs. 30%)을 조정하거나 카카오 채널 리타겟팅 추가 검토. 전환 장벽 분석을 위한 인터뷰 5명 추가 실시. |
| **[가정 3] "향 취향 온보딩 인터뷰 5~7개 질문을 모두 완료하는 사용자가 이탈 없이 첫 레시피를 실행할 것이다"** — 온보딩이 AI 콜드 스타트를 해소하는 유일한 방법이지만, 질문이 많으면 이탈이 발생한다. 온보딩 완주율이 낮으면 AI 추천 품질이 저하되고 첫 인상이 나빠져 DAU 자체가 낮아진다. | **높음** (첫 경험·리텐션 직결, 확신도 낮음) | **퍼널 분석 + 사용성 테스트**: 프로토타입 앱에서 온보딩 각 단계별 이탈률 측정 (n=15, 비관찰 원격 테스트). 이탈 발생 단계 파악 후 Think Aloud 세션 5명으로 정성 보완. 변수: 질문 5개 버전 vs. 7개 버전 비교. | 온보딩 완주율 80% 이상 / 완주 후 첫 레시피 발향 또는 저장 실행률 70% 이상 / 온보딩 소요 시간 중앙값 3분 이하 | 성공 → 현재 질문 구조 확정, A-02 MVP 스펙 동결 / 실패(완주율 60% 미만) → 질문 수 3~4개로 축소하고 나머지 취향 파악은 초기 사용 패턴 관찰로 대체하는 점진적 온보딩(Progressive Onboarding) 전환. |

### MVP UI 우선순위 요약

- **즉시 개발 확정 (Top 5)**: 페어링 → 온보딩 → 발향 제어 → 오일 관리 → 잔량 알림 — 이 5개가 없으면 앱이 동작하지 않거나 첫 경험이 실패한다.
- **핵심 차별화 (6~8위)**: 날씨 AI 추천 + 레시피 저장 + 슬라이더 믹싱 — 이 3개가 Twelve Senses를 경쟁사와 구별한다.
- **이탈 방지 강화 (9~10위)**: 스케줄 + 향 DNA — Q3 이후 리텐션 6개월을 위한 핵심 장치. 데이터 구조는 MVP 설계 단계에서 미리 포함.
